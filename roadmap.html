the roadmap 🗺️

Your step-by-step path
into DevOps.

This is the order I'd follow if I was starting again today. Take it one step at a time — you don't need to rush. Every step builds on the last.

· · · · · · ·
step 01

Linux & the Command Line

🐧

why it matters

Almost everything in DevOps runs on Linux. If you can't navigate the command line, the rest will feel impossible. This is the single most important thing to learn first.

what to learn

  • Navigating files & folders
  • File permissions (chmod, chown)
  • Basic shell scripting
  • SSH into a remote server
  • Package managers (apt, yum)

free resources

  • linuxjourney.com
  • OverTheWire: Bandit
  • Learn Linux TV (YouTube)

Create a Bash script that backs up a folder automatically
step 02

Git & Version Control

🌿

why it matters

You will use Git every single day as a developer or DevOps engineer. It's how teams track changes, collaborate on code, and avoid losing work. Learn it early and it becomes second nature.

what to learn

  • git init, add, commit, push
  • Branching & merging
  • Pull requests on GitHub
  • Resolving merge conflicts
  • .gitignore files

free resources

  • learngitbranching.js.org
  • Git & GitHub Crash Course
  • GitHub Docs (free)

Push a simple project to GitHub with proper commits & a README
step 03

Docker & Containers

🐳

why it matters

Docker changed the way software is built and shipped. Containers let you package your app and all its dependencies so it runs the same on any machine. This is where DevOps starts feeling exciting.

what to learn

  • What containers are (vs VMs)
  • Writing a Dockerfile
  • docker build, run, push
  • Docker Compose for multi-containers
  • Docker Hub

free resources

  • Docker Official Docs
  • TechWorld with Nana (YouTube)
  • Play with Docker (free)

Dockerise a simple web app and push the image to Docker Hub
step 04

CI/CD Pipelines

⚙️

why it matters

CI/CD is the heart of DevOps. Instead of deploying manually, you set up a pipeline that automatically tests, builds, and deploys your code every time you push a change. This is proper DevOps.

what to learn

  • What CI/CD means
  • GitHub Actions basics
  • Writing a YAML workflow file
  • Running tests automatically
  • Deploying on merge to main

free resources

  • GitHub Actions Docs
  • GitHub Actions Full Course
  • TechWorld with Nana (YouTube)

Build a pipeline that auto-deploys your Dockerised app on every push
step 05

Cloud & Kubernetes

☁️

why it matters

The cloud is where real apps live. AWS is the most in-demand cloud platform — and Kubernetes is how teams manage apps at scale. By the time you get here, you'll be job-ready.

what to learn

  • AWS free tier basics (EC2, S3)
  • Terraform for infrastructure
  • What Kubernetes does
  • Minikube locally
  • Helm charts

free resources

  • AWS Free Tier
  • Terraform Tutorials (free)
  • TechWorld with Nana (YouTube)

Deploy an app to AWS EC2 using Terraform — then manage it with Kubernetes

One more thing before you start 💛

Don't try to rush through this. I spent weeks on some of these steps. The goal isn't to go fast — it's to actually understand what you're doing. Build the projects. Break things. Google the errors. That's how it sticks.
